# Professional Development Plan (PDP)

## Introduction

This Professional Development Plan describes the skills I developed during the Secure Systems Architecture module and the areas that I need to improve. The purpose of this plan is to evaluate my progress, identify strengths and weaknesses, and define actions that will help me become a better software developer and team member.

The skills evaluated in this plan are based on the module requirements and include technical skills, teamwork, communication, and problem solving.

---

## Skills Developed During the Module

During this module I developed the following skills:

### 1. Time Management

At the beginning of the module, I found it difficult to manage my time when working on coding tasks and documentation.  
As the project continued, I improved my planning by dividing the work into smaller steps and completing tasks earlier.

**Evidence:**
- Completed the Python prototype step by step
- Submitted assignments before the deadline
- Prepared seminar activities on time

---

### 2. Problem Solving

The project required solving many technical problems, especially during the implementation of the distributed system.

Examples:
- Fixing replay attack issues
- Debugging asynchronous communication
- Improving retry logic
- Reducing energy usage

These problems helped me improve my ability to analyse errors and find solutions.

---

### 3. Communication Skills

Working in a team required clear communication.  
During the seminars and meetings, I shared my ideas about system design and testing.

I also improved my written communication when writing reports, README files, and portfolio entries.

**Evidence:**
- Participated in team discussions
- Prepared presentation slides
- Wrote documentation for the project

---

### 4. Teamwork

The team project helped me understand the importance of cooperation.  
Each member had different tasks, and the project only worked when everyone completed their part.

I learned to listen to feedback from my team and tutor and use it to improve my work.

---

### 5. IT and Programming Skills

This module improved my programming skills in Python.  
I learned how to create simulations, use classes, and implement security features.

Examples of skills learned:

- Object-oriented programming
- Message authentication (HMAC)
- Replay protection
- Testing and logging
- Writing clean and organised code

These skills are important for future software development projects.

---

### 6. Critical Thinking

The module required analysing system security, not only writing code.  
I learned to think about:

- Why a system is vulnerable
- How attacks happen
- How to prevent errors
- How to measure performance

This helped me understand secure system architecture.

---

## Areas for Improvement

Although I improved many skills, there are still areas that need development.

### 1. Better Planning

Sometimes I started coding before finishing the design.  
In the future, I should spend more time on modelling before implementation.

### 2. More Testing

Some problems appeared late because testing started too late.  
In future projects, I should test small parts of the system earlier.

### 3. Confidence in Presentations

I sometimes feel nervous when explaining technical work.  
I want to improve my confidence by practising presentations more often.

---

## Action Plan

To improve my skills, I will:

- Plan projects before coding
- Use diagrams before implementation
- Test code regularly
- Practise explaining technical ideas
- Read more about secure system design
- Work more in team projects

These actions will help me become a better developer.

---

## Conclusion

This module helped me develop important professional skills, including programming, teamwork, and problem solving.  
The Professional Development Plan shows my progress and the steps I will take to improve in the future.

The experience gained in this module will help me in future academic work and real-world software development.
