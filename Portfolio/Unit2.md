# Unit 2 – Modelling Tools and System Design

## Overview

Unit 2 focused on modelling tools used in secure systems architecture.  
The main topic of this unit was the use of UML and SysML as modelling languages for designing complex systems. The unit explained how modelling helps engineers understand system structure before implementation and allows security risks to be identified early in the development process.

We learned that modelling is an important step in secure system design because it allows developers to analyse behaviour, communication, and trust boundaries before writing code.

---

## Seminar and Discussion Forum

In this unit, we participated in a discussion forum comparing UML and SysML.  
We analysed how both modelling languages are used in system engineering and why SysML is often preferred for complex distributed systems.

Two key differences identified were:

- UML is mainly used for software design, while SysML is used for system engineering.
- SysML supports modelling of hardware, software, data, and communication together.

We also read research articles about converting SysML models into executable models.  
This showed how modelling can be connected directly to implementation, which is important for secure system development.

The discussion helped me understand that modelling is not only for documentation but also for testing design ideas before building the system.
